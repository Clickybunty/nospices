NoSpieces - Dokumentation
Projektübersicht
NoSpieces ist eine Anwendung, 
die Rezepte und deren Zutaten verwaltet. 
Sie enthält ein Frontend, 
das mit React entwickelt wurde, 
und ein Backend, 
das mit Node.js und Express aufgebaut ist. 
Die Daten werden in einer MongoDB-Datenbank gespeichert, 
die in AWS gehostet wird.

Architektur
Frontend (React)
Das Frontend ist mit React entwickelt, 
um eine benutzerfreundliche Oberfläche zu bieten. 
Es kommuniziert mit dem Backend über HTTP-Anfragen, 
um Rezepte und Zutaten anzuzeigen und zu verwalten.

Installationsanweisungen:

Navigiere ins Frontend-Verzeichnis:
bash
Code kopieren
cd nospicesfrontend
Erstelle die React-App:
bash
Code kopieren
npx create-react-app nospices
Installiere Axios für die HTTP-Kommunikation:
bash
Code kopieren
npm install axios
Starte die Entwicklungsversion:
bash
Code kopieren
npm start


Backend (Node.js mit Express)
Das Backend bietet APIs, 
um Rezepte und 
Zutaten zu verwalten und mit der Datenbank zu kommunizieren.

Datenmodell
Rezepte
Rezepte sind in einem einfachen JSON-Format gespeichert. 
Jedes Rezept hat eine eindeutige ID und einen Namen. 
Beispiel:

json
{
  "Spaghetti Carbonara": "0001"
}

Zutaten
Die Zutaten sind ebenfalls in einem JSON-Format gespeichert, 
wobei jede Zutat durch eine eindeutige ID referenziert wird. 
Beispiel:

json
{
  "0001": [6002, 5001, 1002, 9002]
}

Die Zahlensequenzen 

(z. B. 6002, 5001, etc.) 

repräsentieren die IDs der Zutaten, 
die für das Rezept benötigt werden.



MariaDB-Datenbank in AWS
Die Anwendung nutzt MariaDB für die Speicherung von Rezepten, 
Zutaten und anderen relevanten Daten. 
Diese Datenbank wird in AWS gehostet, 
um Skalierbarkeit und hohe Verfügbarkeit zu gewährleisten.

MariaDB Setup
Erstellen Sie ein MariaDB-Cluster auf MariaDB Atlas.
Konfigurieren Sie den Cluster und 
generieren Sie eine Verbindung URI.
Fügen Sie die URI in der Backend-Anwendung ein, 
um die Verbindung zur MariaDB-Datenbank herzustellen.
Datenbankstruktur
Rezepte: Eine Sammlung von Rezepten, 
wobei jedes Rezept eine Referenz zu den benötigten Zutaten hat.
Zutaten: Eine Sammlung von Zutaten, 
die durch eine eindeutige ID identifiziert werden.
